Experiments

ControlValueAccessor as a render prop

This is a pretty cool way to reduce the boiler plate of a control value accessor, but it's not super reusable. You can't wrap it in another component and still provide the same level of flexibility of a true form control. In it's current form, it's also not very useful because there's no local state, which isn't necessarily a bad thing, but there's no way to build up form control state from a series of user interactions. You could probably pull it off with some fancy prop getter / state reducer voodoo. Or you could write a component that expects an onChanges function, but then you end up with a lot of extra layers and components.
